# Configuration

## Config file resolution

All Lithos commands accept a `PROJECT` argument which tells Lithos where to look for your config file.
Lithos uses the following rules to find your config file:

1. If the `PROJECT` argument was _not provided_, Lithos treats the current directory as the project directory.
2. If the `PROJECT` argument was _a directory_, Lithos checks that directory for `lithos.yml`, then `lithos.json`, then legacy `mantle.yml`.
3. If the `PROJECT` argument was _a file_, Lithos uses it directly. Explicit config paths can point to either YAML or JSON files.

If multiple discovered config files are present, Lithos uses the same order: `lithos.yml`, then `lithos.json`, then legacy `mantle.yml`.
If no config file is found, Lithos exits with an error that lists those search paths in order.

## File path resolution

Many Lithos config properties require file paths or globs. All file paths should be written relative to the
config file's directory.

For example, with the directory structure:

<FileTree>
  <FileTree.Folder name="project" defaultOpen>
    <FileTree.File name="lithos.yml" />
    <FileTree.File name="game.rbxl" />
    <FileTree.Folder name="marketing" defaultOpen>
      <FileTree.File name="game-icon.png" />
      <FileTree.File name="thumbnail-1.png" />
      <FileTree.File name="thumbnail-2.png" />
    </FileTree.Folder>
  </FileTree.Folder>
</FileTree>

You would use the config:

```yaml filename="project/lithos.yml" {4,6-7,10}
target:
  experience:
    configuration:
      icon: marketing/game-icon.png
      thumbnails:
        - marketing/thumbnail-1.png
        - marketing/thumbnail-2.png
    places:
      start:
        file: game.rbxl
```

Lithos will correctly find the referenced files no matter where you execute Lithos from.

## YAML and JSON syntax

Lithos config files can be written in either YAML or JSON. Directory auto-discovery looks for `lithos.yml`
and `lithos.json`; the legacy fallback remains `mantle.yml`. Config examples in this guide can be switched
between YAML and JSON, and the same schema and field names apply to both.

JSONC and comments are not supported.

To quickly get started with YAML syntax, see "[Learn YAML in Y Minutes](https://learnxinyminutes.com/docs/yaml/)"
or read through the examples in this guide and in the [Examples](https://github.com/siriuslatte/lithos/tree/main/examples)
repo.
